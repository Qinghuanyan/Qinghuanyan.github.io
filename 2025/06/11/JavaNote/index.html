类与对象

  • Java分为两种数据类型：
    • 基本数据类型；
    • 引用数据类型(String和我们创建的所有类)；
  • 类：
    • 属性：这类事物拥有的共有属性；
    • 动作：这一类事物共同执行的功能；
  • 对象：
    • 使用类创建的具体某一个东西；
  • 写代码：
    • 类要用class来定义；
    • 属性：成员变量来描述，直接写在类中的变量；
    • 动作：成员方法，不屑static就是成员方法；
    • 创建对象：
      类 引用 = new 类()；

this关键字

this：

  • 当前类的对象；
  • 可以在方法内部获取到对象中的属性信息；
  • 还可以区分局部变量和成员变量；

构造方法

在创建对象的时候，自动调用的方法；

语法：

注意：

  • 没有返回值这一项；
  • 在我们new的时候，自动调用构造方法；

作用：在创建对象的时候，给对象设置属性信息；

Java会默认自动的送给每一个类一个无参数的构造方法，但是，如果你写了构造方法，系统就不在 赠送了；

构造方法重载

构造方法也是方法，也可以进行重载；

作用：可以有更多的方法去创建对象；

使用this可以访问其他的构造方法；

this()；

static

static.静态

使用p1.country = “民国”;不推荐这样使用静态变量，推荐使用类名去访问静态的内容；

特点：

  • 数据共享
  • 属于类的，并不属于对象
  • 优先于对象产生的
    • 创建对象的过程(简单)：
      1. 静态构造器；
      2. 通用构造器；
      3. 构造方法–>创建对象；

输出证明了是先执行静态构造器，而后执行通用构造器，最后执行构造方法；

由于创建对象的过程在静态内容加载完成之后，所以在静态方法和静态块里不能使用this，静态的内容使用类名去访问；

包和导包

包：本质上就是文件夹

在代码中需要写package 包名；

导包：

  • import 包+类；(import 包*–>会导入包的所有类)；

不需要导包：

  • 在自己的包里；
  • java.lang包下所有的内容都不需要导包(String、System.out.println等)；

访问权限

  1. public 公共的，所有人都可访问；
  2. default 包访问权限，在自己的包内可以随意访问；
  3. private 私有的；

getter和setter

成员变量一般使用private来声明，保护成员变量不被胡乱的赋值；

setter：主要是给成员变量赋值，做一定的保护；

getter：从成员变量中获取数据；

注意：idea里面右键点击“生成”可以一键生成“setter”、“getter”，微调限制即可；

继承

继承：子类可以自动拥有父类中除了私有内容外的其他所有内容；

当出现x是一种y的时候，可以使用x继承y；

super关键字

super：表示父类中的内容；

this：表示自己类中的内容；

用super和this来区分父类和子类中重名的内容；

总结：

  • super可以获取到父类中的内容；
  • 可以调用父类中的构造方法，必须写在子类构造方法的第一行，如果父类的构造是无参的，可以不写，如果父类中有参，必须要写super；

方法的重写

重写：子类对父类中提供的方法进行重新定义；

语法：子类和父类中的方法的声明完全一致；

重写又被称为方法的覆盖；

多态

多态：同一个对象拥有多种形态；

作用：把不同的数据类型进行统一，让程序具有超强的可扩展性；

知识点：

  • 把子类的对象赋值给父类的变量–>向上转型；(猫咪一定是动物，可以将猫咪当作动物来看待)；
    • 缺点：屏蔽掉子类中特有的方法；
  • 把父类的变量转换为子类的变量–>向下转型 ；(但是动物不一定都是猫咪，所以向下转型有风险)；
    • 向下转型可能有风险，Java要求必须要写强制类型转换；(转换之后的数据类型)变量；

以上三种方法均合并为一种，继承自Animal；此处就是利用了多态将Cat、Dog、DaXiang都整合为了动物，都具有吃的功能，但同时猫咪看作动物也失去了抓老鼠的能力；

在这里，三个方法合为一种后，人只需要会喂食动物即可；

final

  • 被final修饰的变量不可以被改变，又被称为常量；
  • 被final修饰的方法不可以被重写；
  • 被final修饰的类不可以被继承；

总结：final是最终式，不可变；

抽象

抽象：现实中不存在的东西；

在Java中：只声明，不实现；

抽象方法：使用abstract来修饰，不可以有方法体，直接用分号结束即可；

抽象类：如果一个类中有抽象方法，这个类必须是一个抽象类；

特点：

  • 抽象类不可以创建对象；
  • 抽象类的子类，必须重写父类中的抽象方法；否则，子类必须也是抽象类；
  • 通过抽象类可以强制的要求子类中必须有哪些方法；

注：抽象类中可以有正常的方法

接口

接口：

  • 实际上是一种特殊的抽象类；
  • 中所有的方法都是抽象方法；
  • 使用interface来声明；
  • 同样具有多态性；
  • 可以把很多不相关的内容进行整合；

注：类只能单继承，接口支持多实现；

特点：

  • 接口中所有的方法都是抽象方法，都是公开的；
  • 接口中所有的变量都是全局静态常量；

成员变量初始值

Java中所有的变量必须先声明，后赋值才能使用；

Java中的成员变量，在创建对象的时候，都会执行一次初始化操作，都会给一个默认值；

基本数据类型默认值都是0，包括Boolean–> false(0即为假)；

引用类型：null，表示空，什么都没有，用来占位的；

object

万事万物皆为对象，所有的东西都是对象；

在Java中所有的类都要继承object，object是一个类，所有类的根；

我们写的类即使不写继承关系，默认也会继承object；

此处的student里面什么都没写，但是仍会有许多的对象，这些就是继承了object获得的；

equals和==

==判断左右两端的数据是否一致；

equals：object类提供的一个方法，用来判断两个对象是否相等(?????)；

两者用法相似，唯一的是equals可以自己重写；

字符串的判断一定要用equals来判断

instanceof关键字

instanceof：判断xxx对象是否是xxx类型的

内存分析

  1. 堆：主要存放对象；
  2. 栈：局部变量，以及基本上数据类型的变量；
  3. 代码区：类和方法；
  4. 数据区：常量池和静态变量；

参数传递的问题

值传递：把变量的值作为参数进行传递；

引用传递：直接把变量作为参数进行传递；

Java使用的是值传递；

异常处理

本质

异常就是错误对象；

  • 编译时异常；
  • 运行时异常；

抛异常

捕获异常，默认由JVM来捕获异常，程序会中断；

分类

  1. RuntimeException，运行时异常(一般不手动处理，出问题了再处理)；
  2. 其他Exception(必须要经过手动处理)；
  3. Error(一般指的是系统级错误)；

处理-try…catch

try{

尝试执行的代码

}catch(Exception e){

处理异常的代码

}finally{

最终的

}

异常的处理-throws和throw

throws表示方法准备要扔出来一个异常；

产生的错误尽可能的自己处理，少向外抛出异常；

throw表示向外抛出异常；

自定义异常

直接继承Exception或者RuntimeException来实现自定义异常(非特定要求别使用)；
